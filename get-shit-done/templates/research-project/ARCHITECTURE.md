<guidelines>

**系统概览：**
- 使用 ASCII 方框字符绘制图表以提高清晰度（├── └── │ ─ 仅用于结构可视化）
- 展示主要组件及其关系
- 不要过度详细 — 这是概念性的，不是实现细节

**项目结构：**
- 具体说明文件夹组织方式
- 解释分组的依据
- 匹配所选技术栈的惯例

**模式：**
- 在有帮助的地方包含代码示例
- 诚实地解释权衡
- 注明何时模式对小项目来说过于复杂

**扩展性考量：**
- 要现实 — 大多数项目不需要扩展到数百万用户
- 聚焦于"什么先崩溃"而非理论极限
- 避免过早优化的建议

**反模式：**
- 特定于此领域
- 包含应该怎么做
- 帮助防止实现过程中的常见错误

</guidelines>
